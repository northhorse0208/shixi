## [1. 两数之和](https://leetcode.cn/problems/two-sum/)

~~~python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        cnt = defaultdict(int)
        for i, num in enumerate(nums):
            other = target - num
            if cnt[other] > 0:
                return [cnt[other]-1, i]
            cnt[num] = i + 1
~~~

第一次错误是因为：cnt[num] = i + 1写成了cnt[other] = i + 1

我的写法是用了defaultdict(int)，因为 `defaultdict(int)` 在遇到不存在的键时会默认返回 `0`，而 Python 列表的第一个元素下标恰好也是 `0`。为了区分“元素不存在”和“元素存在且下标为 0”，不得不在存入时将下标加 1 (`cnt[num] = i + 1`)，并在取出时将下标减 1 (`cnt[other] - 1`)。更优雅的做法是使用普通的字典

~~~python
from typing import List

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        # 使用普通字典作为哈希表：{具体的数字: 对应的下标}
        num_map = {} 
        
        for i, num in enumerate(nums):
            other = target - num
            # 直接检查配对的数字是否在字典的键中
            if other in num_map:
                # 如果在，直接返回存好的下标和当前的下标
                return [num_map[other], i]
            
            # 如果不在，将当前数字和真实下标存入字典
            num_map[num] = i
            
        return []
~~~



## [49. 字母异位词分组](https://leetcode.cn/problems/group-anagrams/)

思路：将列表中每个字符串先排序下，这个作为哈希key

~~~py
class Solution:
    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        d = defaultdict(list)
        for s in strs:
            sorted_s = ''.join(sorted(s)) # 易错点1
            d[sorted_s].append(s)
        return list(d.values())# 易错点2
~~~

易错点1：`sorted()` 函数可以接收任何**可迭代对象（iterable）**。因为字符串是由字符组成的序列，所以它是可以迭代的。<span style='color:blue'>当你对一个字符串使用 `sorted()` 时，Python 会把字符串拆成一个个单独的字符，按照 ASCII/Unicode 码进行排序，最后**返回一个由字符组成的列表（List）**，而不是一个排好序的字符串。</span>

易错点2：外面要套一层list。当你调用字典的 `d.values()` 时，它返回的**并不是一个真正的列表**，而是一个叫做 **“字典视图对象”（Dictionary View Object）** 的东西，具体类型叫 `dict_values`。套一层 `list()` 就是强制类型转换，把 `dict_values` 视图里面包含的元素提取出来，打包成一个真正的标准列表返回。

~~~python
class Solution:
~~~

~~~python
        """
        Do not return anything, modify nums in-place instead.
        """
        left = 0
        for right, num in enumerate(nums):
            if num != 0:
                nums[right], nums[left] = nums[left], nums[right]
                left += 1
~~~

## [11. 盛最多水的容器](https://leetcode.cn/problems/container-with-most-water/)

left，right分别初始化为0，长度减一，从两边往中间夹逼。

注意：这个题本质上是看left和right的高度最小值乘以两个坐标相减。所以，最开始令坐标相减值最大，也就是left，right分别在两边。然后如果缩减了坐标值，还想让盛水更多的话，只能缩小left和right中高度比较小的那个的值，看看还能不能找到更大的最小值。

~~~python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        ans = -inf
        while left < right:
            ans = max(ans, min(height[left], height[right]) * (right - left))
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        return ans  
~~~

## [42. 接雨水](https://leetcode.cn/problems/trapping-rain-water/)

介绍两种方法：前后缀分解法、相向双指针方法

**方法1：前后缀分解法**

主要思路是假设每个坐标处放了一个宽度是1的桶，然后看这个桶的前后缀最大值的最小值再减去这个桶的高度就是这个桶这儿最多能装多少水。（**注意：如果前后缀最大值都小于这个桶本身的高度，那么就得直接令这个桶能接的水是0，而不是继续减导致这个桶能盛的水是负数，我的代码第一遍就犯了这个错误**）然后把所有桶最多能装的水加起来就行了

~~~python
class Solution(object):
    def trap(self, height):
        """
        :type height: List[int]
        :rtype: int
        """
        n = len(height)
        suf_max = [0] * n
        pre_max = [0] * n
        for i in range(n-2, -1, -1):
            suf_max[i] = max(suf_max[i+1], height[i+1])
        ans = 0
        for i in range(1, n):
            pre_max[i] = max(pre_max[i-1], height[i-1])
            ans += min(pre_max[i], suf_max[i]) - height[i] if min(pre_max[i], suf_max[i]) > height[i] else 0
        return ans
~~~

第一次犯的错误：ans += min(pre_max[i], suf_max[i]) - height[i] if min(pre_max[i], suf_max[i]) > height[i] else 0 没有加后面的 - height[i] if min(pre_max[i], suf_max[i]) > height[i] else 0。-height[i]，这个减去桶的初始高度是必要的。判断前后缀最大值中的最小值是否大于这个桶也是必要的。

**方法2：相向双指针方法**

还是用木桶来理解。初始令left，right分别指向两端，然后维护left对应桶的前缀最大值和right对应桶的后缀最大值。假设，left对应桶的前缀最大值 < right对应桶的后缀最大值，那么left对应桶的后缀最大值一定大于等于这个right对应桶的后缀最大值，所以他left对应的这个桶能盛多少水可以知道了。left对应的坐标就可以移动了。同理，left对应桶的前缀最大值 > right对应桶的后缀最大值的话，right的坐标可以移动了。

**总之：双指针，移动对应的后缀比较小的那个指针。**

代码编写使用的是灵神那套思路，就是计算前后缀需要跟自身考虑

~~~python
class Solution(object):
    def trap(self, height):
        """
        :type height: List[int]
        :rtype: int
        """
        n = len(height)
        left, right = 0, n - 1 
        ans = 0
        pre_max, suf_max = 0, 0
        while left <= right:
            pre_max = max(pre_max, height[left])
            suf_max = max(suf_max, height[right])
            if pre_max < suf_max:
                ans += pre_max - height[left]
                left += 1
            else:
                ans += suf_max - height[right]
                right -= 1
        return ans
~~~

易错点：while left <= right: 不能是while left < right: ， 因为left和right相等时那个桶也可能需要装水

~~~python
class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        cnt = Counter(p)
        left = 0
        ans = []
        for right, c in enumerate(s):
            cnt[c] -= 1
            while cnt[c] < 0:
                cnt[s[left]] += 1
                left += 1
            if right - left + 1 == len(p):
                ans.append(left)

        return ans
~~~

## [239. 滑动窗口最大值](https://leetcode.cn/problems/sliding-window-maximum/)

**本质单调队列题目**

1. 维护一个单调递减的队列，这个队列里存放当前窗口中单调递减数组**的下标**
2. 遍历数组，如果该值大于队尾元素，就pop一下队尾元素，直到队列为空或者队尾元素大于该值了。如果队首元素-当前遍历的下标=k，证明这次移动窗口左边掉出窗口的元素是队首元素，因此需要将队首元素pop出去。如果遍历的下标大于k-1了，证明可以取队首元素作为最大值加入ans列表中了。
3. 下面有gpt关于该代码的详细解释，如果上面我自己写的看不懂的话

~~~python
class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        ans = []
        q = deque()
        for i, num in enumerate(nums): # 解释点1
            while q and nums[q[-1]] < num:
                q.pop()
            q.append(i)
            if i - q[0] == k:
                q.popleft()
            if i >= k - 1:
                ans.append(nums[q[0]])
        return ans
~~~

解释点1：写 `<=` 还是 `<` 没有任何区别，都能 100% 得到正确答案。但是，从**极致的性能和空间优化**角度来看，**写 `<=` 会更好一点点。**

> gpt具体解释：
>
> 可以用我们刚才“排队”的例子来直观理解这两者的区别。假设遇到了一串连续相等的数字，比如 `nums = [5, 5, 5, 5]`，窗口大小 `k = 3`：
>
> - **当你写 `<=`（小于等于）时：** 它的逻辑是：“新来的年轻人 `x`，只要能力**不比你差（大于或等于你）**，那你这个老员工的利用价值就没了（毕竟你还比人家老），可以直接退休了。” 在遇到连续的 `[5, 5, 5]` 时，后面的 5 会把前面的 5 踢掉。队列里**永远只会保留最新进来的那个 5 的索引**。 *优势：* 极其节省空间。不管窗口里有多少个重复的当前最大值，队列里只存一个最新鲜的。
> - **当你写 `<`（严格小于）时：** 它的逻辑是：“新来的年轻人必须**严格比你强**，才能把你踢走；如果和你一样强，那大家就一起在队列里排着吧。” 在遇到连续的 `[5, 5, 5]` 时，前面的 5 不会被踢走，它们会挨个排在队列里：比如 `q = [0, 1, 2]`（对应的值都是 5）。直到排在最前面的 5 因为太老了（滑出窗口），被下面的代码 `q.popleft()` 自然淘汰。 *劣势：* 如果窗口 `k` 非常大（比如 k=10000），并且数组里恰好有一大段全都是相同的数字，那双端队列 `q` 里面最多可能会积压 10000 个元素，浪费了不必要的内存。
>
> **总结：** 这两个符号的区别仅仅在于“是否允许队列里存在相同大小的值”。
>
> - 写 `<` 逻辑上没毛病，因为虽然队列里存了多个相同的值，但排在最前面的那个（队头）确实还是当前窗口的最大值，取 `nums[q[0]]` 依然是正确的。
> - 写 `<=` 是一种更为极致的写法，把冗余的相等元素也提前踢掉，让队列保持绝对的精简。

> 关于本题的代码，gpt更详细的解释如下:
>
> 在这个算法中，双端队列 `q` 里存的不是数字本身，而是数字的**索引（下标）**。 队列始终保持一个特性：**从队头到队尾，对应的数字是严格递减的。** 这样一来，队头（`q[0]`）永远是当前窗口里的最大值。
>
> 我们把滑动窗口看作是一个时代的变迁，来看看代码是如何运作的：
>
> 1. **“年轻且更强” 的淘汰机制 (`while` 循环)：** 每当遍历到一个新数字 `x`（新入职的年轻人），我们就去跟队列尾部的人比。如果新来的 `x` 比队尾对应的数字还要大（或一样大），那么队尾的人就可以被“开除”了（`q.pop()`）。因为只要 `x` 还在窗口里，前面那些比他小的人就**永远不可能**成为窗口最大值。
> 2. **新老交替入队 (`q.append`)：** 所有新来的 `x` 的索引 `i`，最后都会进入队列排队，毕竟它年轻，万一前面更强的人“退休”了，它还是有机会当最大的。
> 3. **“退休” 机制 (`if i - q[0] == k`)：** 虽然队头的人最强（当前最大值），但他也是最老的。如果当前索引 `i` 减去队头索引 `q[0]` 等于窗口大小 `k`，说明队头这个最强的人已经被滑出了窗口（退休了），必须把他从队头踢掉（`q.popleft()`）。
> 4. **颁发奖项 (`if i >= k - 1`)：** 当窗口刚开始滑动时，还没满 `k` 个人，不能颁发“当前最大值”奖。只有当索引走到 `k-1`（也就是第一个窗口成型时）及以后，我们每次都要把队头最强的人记录进答案数组 `ans` 中。



## [76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/)

**方法1**   灵神题解中的最优解法

1. 初始化“欠账本” (统计 t)：维护一个diff = defaultdict(int)，统计目标串 `t` 中每个字母的数量，全部记为负数（代表你**欠**了这么多字母）。同时，记录 `t` 中一共包含多少**种**不同的字母，存入变量 `kinds`。

2. 准备双指针和“达标计数器”：初始化左指针 `left = 0`，以及用于记录最终最短子串的两个端点 `ans_left` 和 `ans_right`。 准备一个核心变量 `ge_cnt = 0`（达标计数器），用来记录当前窗口内，**有几种字母的“欠账”已经被还清了**（数量从负数变成了 >= 0）。

3. 右指针主动右移（进窗口还账）：遍历字符串 `s`，把右指针指向的字母不断加入窗口，即在 `diff` 字典中把该字母的数量 `+1`（相当于还了一个字母）。

   **关键判断：** 如果这个字母加完之后，数量**刚好等于 0**（说明这种字母不仅补齐了，而且刚刚好达标），就把达标计数器 `ge_cnt` 增加 1。

   只要 `ge_cnt == kinds`（说明所有种类的字母都达标了，当前窗口已经包含了 `t`），就开启一个 `while` 循环尝试缩小窗口：

   * **记录答案：** 看看当前窗口长度是不是比历史最短记录还要短，如果是，就更新 `ans_left` 和 `ans_right`。
   * **准备移出左端点：** 拿左指针指向的字母看看，如果它现在的数量**刚好是 0**，说明一旦把它踢出去，这种字母就不达标了，必须把 `ge_cnt` 减 1。
   * **正式移出：** 在 `diff` 字典中把这个左端点字母的数量 `-1`，并且左指针往右走一步（`left += 1`）。
   * (这个过程会一直循环，直到踢出去某个关键字母导致 `ge_cnt` 不等于 `kinds`，才会退出循环，继续执行第三步)

4. 返回最终结果：整个遍历结束后，检查一下记录的端点 `ans_left`。如果它还是初始的负值，说明压根没找到过符合条件的子串，返回空字符串 `""`；否则，就根据记录的左右端点，从 `s` 中截取并返回那段最短的子串。

~~~py
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        # 注：defaultdict 比 Counter 快
        diff = defaultdict(int)  # 窗口每种字母个数 - t 每种字母个数
        for c in t:
            diff[c] -= 1
        kinds = len(diff)  # t 中有 kinds 种不同的字母

        ans_left, ans_right = -1, len(s)
        ge_cnt = 0  # 窗口内有 ge_cnt 种字母的出现次数 >= t 中相应字母的出现次数
        left = 0

        for right, c in enumerate(s):  # 移动子串右端点
            diff[c] += 1  # 右端点字母移入子串
            if diff[c] == 0:  # 原来窗口内 c 的出现次数比 t 的少，现在一样多
                ge_cnt += 1  # 从 < 变成 >=

            while ge_cnt == kinds:  # 涵盖：所有字母的出现次数都是 >=
                if right - left < ans_right - ans_left:  # 找到更短的子串
                    ans_left, ans_right = left, right  # 记录此时的左右端点

                x = s[left]  # 左端点字母 # 易错点1
                if diff[x] == 0:
                    # x 移出窗口之前，检查出现次数，
                    # 如果窗口内 x 的出现次数和 t 一样，
                    # 那么 x 移出窗口后，窗口内 x 的出现次数比 t 的少
                    ge_cnt -= 1  # 从 >= 变成 <
                diff[x] -= 1  # 左端点字母移出子串
                left += 1

        return "" if ans_left < 0 else s[ans_left: ans_right + 1]
~~~

易错点1：注意这里是左端点字母，第一次错误是直接用了右端点字母c

**方法2   ** 灵神的次优解法，用的Counter，先不补充了

## [56. 合并区间](https://leetcode.cn/problems/merge-intervals/)

1. 关键一步：根据列表中每个区间的左端点从小到大排序 ```intervals.sort(key = lambda p:p[0])``` 复习一下sort中key函数的涉及
2. 维护一个ans。先把第一个区间放进去
3. 遍历排序好区间，如果新区间的左端点小于等于ans最后一个区间的右端点，把ans最后一个区间的右端点改为原值和新区间右端点中的最大值；如果新区间的左端点大于ans最后一个区间的右端点，直接加入ans即可

~~~python
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key = lambda p:p[0])
        ans = []
        ans.append(intervals[0])
        for p in intervals:
            if p[0] <= ans[-1][1]:
                ans[-1][1] = max(ans[-1][1], p[1])
            else:
                ans.append(p)
        return ans
~~~



## [189. 轮转数组](https://leetcode.cn/problems/rotate-array/)

举例：```[1, 2, 3, 4, 5, 6, 7]```向右转3轮就是```[5, 6, 7, 1, 2, 3, 4]```

记住结论：先把```[1, 2, 3, 4, 5, 6, 7]```翻转一下变为```[7, 6, 5, 4, 3, 2, 1]```，然后前三个翻转一下，后四个翻转一下

~~~python
class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        def reverse(i, j):
            while i < j:
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
                j -= 1
        reverse(0, len(nums)-1)
        k = k % len(nums) # 易错点1
        reverse(0, k-1)
        reverse(k, len(nums)-1)
~~~

易错点1：注意这里先模一下，否则k可能大于了整个数组的长度

## [54. 螺旋矩阵](https://leetcode.cn/problems/spiral-matrix/)

1. 定义top bottom left right 记录当前矩阵的动态边界（初始化top 和 left是0，bottom是行数-1，right是列数-1），也就是说如果往右走到最后了，那么top就需要减加1了，其他边界值调整自己想一想。
2. 定义一个while True的循环，依次执行 右行 下行 左行 上行 的操作。执行完每个子操作之后，将某个边界做变化，然后判断一下top是不是大于bottom了，或者left是不是大于right了，如果是的话，那么证明已经没有元素可以遍历了，就可以退出了。

~~~py
class Solution:
    def spiralOrder(self, matrix: List[List[int]]) -> List[int]:
        if not matrix:
            return []
        ans = []
        top, bottom, left, right = 0, len(matrix)-1, 0, len(matrix[0])-1
        while True:
            for i in range(left, right + 1):
                ans.append(matrix[top][i])
            top += 1
            if top > bottom: break
            
            for i in range(top, bottom+1):
                ans.append(matrix[i][right])
            right -= 1
            if left > right: break

            for i in range(right, left-1, -1):
                ans.append(matrix[bottom][i])
            bottom -= 1
            if top > bottom: break

            for i in range(bottom, top-1, -1):
                ans.append(matrix[i][left])
            left += 1
            if left > right: break

        return ans
~~~



## [48. 旋转图像](https://leetcode.cn/problems/rotate-image/)

**做法**1：直观做法：分块

![image-20260514201417883](C:\Users\29743\AppData\Roaming\Typora\typora-user-images\image-20260514201417883.png)

根据上面的五角星的转换，可以得到一个规律，如果第一个五角星的坐标为（i，j），第二个则是（j，n-1-i）也即第二个五角星的横坐标是第一个五角星的纵坐标，第二个五角星的纵坐标与第一个五角星横坐标之和为n-i。第三个之于第二个也是这样的关系，横坐标等于第二个的纵坐标，纵坐标等于他的n-1-第二个的横坐标，也即（n-1-i，n-1-j）。同理，推出第四个坐标为（n-1-j，i）。

~~~python
class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        """
        Do not return anything, modify matrix in-place instead.
        """
        m = len(matrix)
        for i in range(m // 2):
            for j in range((m + 1) // 2):
                matrix[i][j], matrix[j][m-1-i], matrix[m-1-i][m-1-j], matrix[m-j-1][i] = matrix[m-j-1][i], matrix[i][j], matrix[j][m-1-i], matrix[m-1-i][m-1-j]
~~~

**做法2**：灵神做法

根据上面的思路可以知道，旋转90°就是把原坐标（i，j）转到（j，n-1-i）

$(i,j) \to (j,n-1-i)$ 可以通过两次翻转操作得到：

$$(i,j) \xrightarrow{\text{转置}} (j,i) \xrightarrow{\text{行翻转}} (j,n-1-i)$$

1. 转置：把矩阵按照主对角线翻转，位于 $(i,j)$ 的元素去到 $(j,i)$。
2. 行翻转：把每一行翻转，位于 $(j,i)$ 的元素去到 $(j,n-1-i)$。

示例 1 的操作过程如下：

$$\begin{bmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \\ 7 & 8 & 9 \end{bmatrix} \xrightarrow{\text{转置}} \begin{bmatrix} 1 & 4 & 7 \\ 2 & 5 & 8 \\ 3 & 6 & 9 \end{bmatrix} \xrightarrow{\text{行翻转}} \begin{bmatrix} 7 & 4 & 1 \\ 8 & 5 & 2 \\ 9 & 6 & 3 \end{bmatrix}$$

> 注：一般地，把一个点绕 $O$ 旋转任意 $\theta$ 角度，可以通过两个翻转操作实现。要求这两条翻转的对称轴，交点为 $O$ 且夹角为 $\frac{\theta}{2}$。对于本题，每个元素需要绕矩阵中心顺时针旋转 90°，这可以通过关于主对角线翻转，关于垂直中轴翻转实现。这两条对称轴的交点为矩阵中心，且夹角为 45°。

~~~py
class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        n = len(matrix)
        # 第一步：转置
        for i in range(n):
            for j in range(i):  # 遍历对角线下方元素
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

        # 第二步：行翻转
        for row in matrix:
            row.reverse() # 易错点1
~~~

易错点1：reverse()函数是python中列表自带的函数，就是原地翻转列表

**做法3**：终极压缩，只用一层循环

需要把遍历顺序调整为遍历对角线上方元素，这样每行遍历完后，这一行的元素后面不会再访问到，可以直接做行翻转。

~~~python
class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        n = len(matrix)
        for i, row in enumerate(matrix):
            for j in range(i + 1, n):  # 遍历对角线上方元素，做转置
                row[j], matrix[j][i] = matrix[j][i], row[j]
            row.reverse()  # 行翻转
~~~



## [240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/)

灵神解法：矩阵的四个角是最特殊的，左上角的值最小，右下角的值最大，另两个角的值居中。对于有序数据，从右上角或者左下角这种中间值开始，可以帮助我们排除不含答案的行列。

1. 初始化i，j表示右上角的值
2. while一个循环，要求i 不能出下界，j又不能小于0：在每一次循环中，查看target和当前矩阵右上角的关系，如果相等，就返回。如果当前矩阵右上角小于target，证明这一行都不会有target，可以删除这一行，也就是令i加1，缩减矩阵的范围。同理，如果矩阵右上角大于target，证明这一列都不会有target，可以删除这一列，也就是说j减1。
3. while结束了，还没找到，就是找不到了

~~~python
class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        m, n = len(matrix), len(matrix[0])
        i, j = 0, n - 1  # 从右上角开始
        while i < m and j >= 0:  # 还有剩余元素
            if matrix[i][j] == target:
                return True  # 找到 target
            if matrix[i][j] < target:
                i += 1  # 这一行剩余元素全部小于 target，排除
            else:
                j -= 1  # 这一列剩余元素全部大于 target，排除
        return False
~~~

## [148. 排序链表](https://leetcode.cn/problems/sort-list/)

大体上是灵神的题解做法

**方法一**：归并排序

需要构造三个函数

1. 函数1：传入头节点，找到**链表的中间结点** *head*2 的**前一个节点**，并断开 *head*2 与其前一个节点的连接。这样我们就把原链表均分成了两段更短的链表。返回中间节点
2. 函数2：传入两个排好序的链表的头节点，将他们整合成一个链表返回
3. 函数3：排序函数，传入链表的头节点。如果当前链表为空，或者只有一个节点，直接返回即可。如果节点大于一个，就先调用函数1将链表切分，然后分别对这两个子链表递归调用函数3进行排序。然后将这两个排好序的链表调用函数2合并到一起，返回即可。

~~~python
class Solution:
    # 链表的中间结点。 相比于普通的求中间节点，这个多余的一部操作是要维护上一步的slow，这样方便将第一段链表末尾节点的next设置为None，达到切分的效果
    def middleNode(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow = fast = head
        while fast and fast.next:
            pre = slow  # 记录 slow 的前一个节点
            slow = slow.next
            fast = fast.next.next
        pre.next = None  # 断开 slow 的前一个节点和 slow 的连接
        return slow

    # 21. 合并两个有序链表（双指针）
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        cur = dummy = ListNode()  # 用哨兵节点简化代码逻辑
        while list1 and list2:
            if list1.val < list2.val:
                cur.next = list1  # 把 list1 加到新链表中
                list1 = list1.next
            else:  # 注：相等的情况加哪个节点都是可以的
                cur.next = list2  # 把 list2 加到新链表中
                list2 = list2.next
            cur = cur.next
        cur.next = list1 if list1 else list2  # 拼接剩余链表
        return dummy.next

    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 如果链表为空或者只有一个节点，无需排序
        if head is None or head.next is None:
            return head
        # 找到中间节点 head2，并断开 head2 与其前一个节点的连接
        # 比如 head=[4,2,1,3]，那么 middleNode 调用结束后 head=[4,2] head2=[1,3]
        head2 = self.middleNode(head)
        # 分治
        head = self.sortList(head)
        head2 = self.sortList(head2)
        # 合并
        return self.mergeTwoLists(head, head2)
~~~

~~~python
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:

        cur = dummy_node
        heap = []
        for list in lists:
            if list is not None:

        while heap:
            node = heappop(heap)
            if node.next:
                heappush(heap, node.next)

            cur = cur.next
        return dummy_node.next
~~~

## [146. LRU 缓存](https://leetcode.cn/problems/lru-cache/)

1. 这个题节点需要使用双向链表，至于为什么，往下看
2. 由于本体中所有的key是不会重复的，并且题目要求get操作需要o(1)的复杂度完成，因此可以用一个key to node的哈希表，这样知道key之后，直接访问这个哈希表就能知道对应的node

4. 本题需要加一个函数 remove()函数：传入需要remove的key，根据这个key在key to node中查找需要remove的节点，因此要知道这个节点的前一个节点，因此才需要使用双向链表。同时注意将key to node中对应的key删除掉
5. put函数：首先查看要put的这个key是否在key to node哈希表中，如果在，就需要先 remove(key) 处理；把这个key 和 value 创建一个新node，插入到dummynode的next，同时修改key to node中该key对应的node；根绝key to node的长度判断是否超过了LRU容量，如果超过了，找到dummynode的pre节点的key，然后remove
6. get函数：查看key是否在key to node中，如果不在，返回-1，如果在的话，先记录value。然后remove这个节点，在put这个节点。（因为LRU中，get算作访问了一次，他就要放到开头）

~~~python
class ListNode:
    def __init__(self, key=0, val=0, next=None, pre=None):
        self.key = key
        self.val = val
        self.next = next
        self.pre = pre

class LRUCache:

    def __init__(self, capacity: int):
        self.capacity = capacity
        self.dummyNode = ListNode()
        self.dummyNode.next = self.dummyNode
        self.dummyNode.pre = self.dummyNode
        self.keytonode = {}

    def get(self, key: int) -> int:
        if key in self.keytonode:
            value = self.keytonode[key].val
            self.remove(key)
            self.put(key, value)
            return value
        return -1

    def put(self, key: int, value: int) -> None:
        if key in self.keytonode:
            self.remove(key)

        new_node = ListNode(key=key, val=value, pre=self.dummyNode, next=self.dummyNode.next)
        self.keytonode[key] = new_node
        new_node.next.pre = new_node
        self.dummyNode.next = new_node

        if len(self.keytonode) > self.capacity: # 易错点1
            self.remove(self.dummyNode.pre.key)


    def remove(self, key: int) -> None:
        removedNode = self.keytonode.pop(key) # 易错点2
        removedNode.pre.next = removedNode.next
        removedNode.next.pre = removedNode.pre
~~~

思路1：在函数内部创立一个ans。然后创建一个子函数，这个子函数只做遍历，然后中间把值加入ans列表中。**当遍历的节点是空节点时，直接返回None即可**。

最后要**执行下这个函数**，然后返回ans即可。

~~~python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        ans = []
        def dfs(node):
            if not node:
                return
            dfs(node.left)
            ans.append(node.val)
            dfs(node.right)
        dfs(root)
        return ans
~~~

思路2：不用子函数了，当前这个函数返回的就是最终答案，每个函数返回的都是当前根节点的树的中序遍历的结果。**所以当遍历的节点是空节点时，需要返回一个空列表，而不是None**

~~~python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        if root is None:
            return []
        left = self.inorderTraversal(root.left)
        right = self.inorderTraversal(root.right)
        return left + [root.val] + right
~~~

## [108. 将有序数组转换为二叉搜索树](https://leetcode.cn/problems/convert-sorted-array-to-binary-search-tree/)

整数数组已经排好序了。将数组分为两份，但注意留出中间的值作为根节点的值，这两份分别递归执行本函数。最后返回一个TreeNode实例，他的值就是预留出来中间的值，左右子树就是刚才递归得到的。边界情况也很清晰了，如果传入的节点是空节点，直接返回None即可。

关于预留出来中间值的操作，可以这样理解。当n为奇数，比如7时，7//2 = 3，所以就是[:3] (代表取前三个数) [4:] (代表取后两个数) ，下标为3就是中间预留的那个值。当n为偶数，比如6时，6//2 = 3，所以就是[:3] 取前三个数，[4:]取后两个数，下标为3的就是中间预留的那个。所以不管奇偶，在分片的时候都是按照m = 数字长度 // 2，然后m就是作为最后的根节点的值，两个要递归的区间分别是[:m]  [m+1:]

~~~py
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def sortedArrayToBST(self, nums: List[int]) -> Optional[TreeNode]:
        if not nums:
            return None
        m = len(nums) // 2
        left = self.sortedArrayToBST(nums[:m])
        right = self.sortedArrayToBST(nums[m+1:])
        return TreeNode(nums[m], left, right)
~~~

~~~python
    def qianxu(self, node, left, right):
~~~

## [230. 二叉搜索树中第 K 小的元素](https://leetcode.cn/problems/kth-smallest-element-in-a-bst/)

大致就是中序遍历可以得到从小到大的排列。

**方法1**：记录答案

1. 先设置ans，代表要存的答案
2. 创建子函数，单纯的中序遍历树。但是每次遍历完左边之后，把k减去1，如果k == 0 了，证明档前这个数字就是答案了，令ans等于她就行了。然后遍历右子树。子函数中记得将ans 和 k做nonlocal处理。
3. 边界值有意思，就是除了传统的当前节点如果是None，就跳过外，多加一个，如果k < 0了，也可以直接return了，作为一个剪枝。
4. 记得执行一次子函数

~~~python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        ans = 0
        def dfs(node):
            nonlocal ans, k
            if not node or k < 0:
                return 
            dfs(node.left)
            k -= 1
            if k == 0:
                ans = node.val
            dfs(node.right)
        dfs(root)
        return ans
~~~

## [114. 二叉树展开为链表](https://leetcode.cn/problems/flatten-binary-tree-to-linked-list/)

灵神的题解

**做法1：头插法**

采用头插法构建链表，也就是从节点 6 开始，在 6 的前面插入 5，在 5 的前面插入 4，依此类推。

为此，要按照 6→5→4→3→2→1 的顺序访问节点。如何遍历这棵树，才能实现这个顺序？

按照**右子树 - 左子树 - 根**的顺序 DFS 这棵树。（这个顺序既不是前序，也不是中序，也不是后序，而是前序倒过来）

DFS 的同时，记录当前链表的头节点为 *head*。一开始 *head* 是空节点。

1. 先在类下设置head为None
2. 递归右子树，递归左子树，处理根节点
3. 将根节点左子树指向空，右子树指向head
4. head需要移向当前链表的头节点，也就是root
5. 函数的边界条件就是如果传进来的节点是空节点，就直接返回即可

~~~python
class Solution:
    head = None
    def flatten(self, root: Optional[TreeNode]) -> None:
        """
        Do not return anything, modify root in-place instead.
        """
        if not root:
            return 
        self.flatten(root.right) # 易错点1
        self.flatten(root.left)
        root.left = None
        root.right = self.head
        self.head = root
~~~

易错点1：这里一定是先递归右子树，后递归左子树啊。不能反了

**方法2：分治法**    方法一需要用到一个在 DFS 之外的变量 *head*，能否只在 DFS 中解决呢？

需要改变下本函数的返回，本函数再执行完之后，需要返回节点的右子树的最后一个，如果右子树最后一个为空，就需要返回左子树最后一个，如果左子树最后一个也为空，就需要返回跟节点，根节点一定不为空，因为为空的话上面就已经处理了。

~~~py
class Solution:
    def flatten(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        if root is None:
            return None
        left_tail = self.flatten(root.left)
        right_tail = self.flatten(root.right)
        if left_tail:
            left_tail.right = root.right  # 左子树链表 -> 右子树链表
            root.right = root.left  # 当前节点 -> 左右子树合并后的链表
            root.left = None
        return right_tail or left_tail or root
~~~

## [105. 从前序与中序遍历序列构造二叉树](https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/)

用递归做

1. 把前序第一个数拿出来，记为值root，找到中序中他在第几位，记为index
2. 也就是说中序中[:index] [index+1:]是左右子树的中序。前序中，[1:index+1]类比中序中第一个序列取值都往后移动一位，是左子树的前序；但是右子树的前序就不像中序中间隔一个了，是紧挨着的，所以就是[index+1:]
3. 边界条件也就清晰了。就是当前序中没有数字的时候，返回个None即可

~~~py
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        if not preorder:
            return None
        root = preorder[0]
        index = inorder.index(root)
        left_pre = preorder[1:1+index]
        right_pre = preorder[1+index:]
        left_in = inorder[:index]
        right_in = inorder[index+1:]
        return TreeNode(root, self.buildTree(left_pre, left_in), self.buildTree(right_pre, right_in))
~~~

5. cnt[s] += 1，然后递归调用dfs的左右子树
6. 记得回复现场，就是把cnt[s] -= 1
7. 执行dfs(root, 0)即可。

~~~py
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        # key：从根到 node 的节点值之和
        # value：节点值之和的出现次数
        # 注意在递归过程中，哈希表只保存根到 node 的路径的前缀的节点值之和
        cnt = defaultdict(int)  
        cnt[0] = 1
        ans = 0

        # s 表示从根到 node 的父节点的节点值之和（node 的节点值尚未计入）
        def dfs(node: Optional[TreeNode], s: int) -> None:
            if node is None:
                return

            nonlocal ans
            s += node.val
            # 把 node 当作路径的终点，统计有多少个起点
            ans += cnt[s - targetSum]

            cnt[s] += 1
            dfs(node.left, s)
            dfs(node.right, s)
            cnt[s] -= 1  # 恢复现场（撤销 cnt[s] += 1）

        dfs(root, 0)
        return ans
~~~

~~~python
    def lowestCommonAncestor(self, root: 'TreeNode', p: 'TreeNode', q: 'TreeNode') -> 'TreeNode':

        # 如果越界了，或者当前节点刚好就是我们要找的 p 或 q，直接返回当前节点
        if root == None or root == p or root == q:

        right = self.lowestCommonAncestor(root.right, p, q)

        # 如果左右子树都各自找到了目标，说明 p 和 q 刚好分居在当前 root 的两侧

        if left and right:
            return root
        
        # 如果只有左子树找到了，说明目标（或者祖先）在左边，把左边的结果向上返回
        if left:
            return left
        # 否则，要么是在右边找到了，要么都没找到（right 为 None），都直接返回 right
        else:
            return right
~~~

## [124. 二叉树中的最大路径和](https://leetcode.cn/problems/binary-tree-maximum-path-sum/)

大体思路就是，定义一个递归函数，这个函数是传入一个节点，最后返回以这个节点为起点，往下走能得到的最大链的路径和。然后递归执行这个函数，切记，他的值等于左节点或者右节点执行该函数后中的最大值，但应该要大于0，否则只是用本节点即可。在递归的过程中，同步计算一下以该节点作为拐点的最大路径的长度即可。

~~~python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        ans = -inf
        def dfs(node: Optional[TreeNode]) -> int:
            if node is None:
                return 0  # 没有节点，和为 0
            l_val = dfs(node.left)  # 左子树最大链和
            r_val = dfs(node.right)  # 右子树最大链和
            nonlocal ans
            ans = max(ans, l_val + r_val + node.val)  # 两条链拼成路径
            return max(max(l_val, r_val) + node.val, 0)  # 当前子树最大链和（注意这里和 0 取最大值了）
        dfs(root)
        return ans
~~~

~~~python
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        g = [[] for _ in range(numCourses)]
        for a, b in prerequisites:
            g[b].append(a)
        color = [0] * numCourses
        
        def dfs(i):
            color[i] = 1
            for x in g[i]:
                if color[x] == 1:
                    return True
                if color[x] == 0:
                    if dfs(x):
                        return True
            color[i] = 2
            return False
            
        for i in range(numCourses):
            if color[i] == 0:
                if dfs(i):
                    return False
        return True
~~~

易错点1：为什么这里是```range(i, len(nums))```而不是```range(i + 1, len(nums))```？

> 简单来说：
>
> - **`i` 代表：“当前这一步，我被允许从哪个位置开始挑选数字”。**
> - **`j` 代表：“当前这一步，我实际挑中了哪个位置的数字”。**
>
> 下面为你详细拆解为什么这里必须是 `range(i, len(nums))`。
>
> **1. 为什么循环必须从 `i` 开始？ (`range(i, ...)`)**
>
> 当你调用 `dfs(i)` 时，参数 `i` 传达的信息是：**“前面的数字都已经考虑过了，从现在开始，你可以从索引 `i` 往后挑。”**
>
> 既然你被允许从索引 `i` 开始挑，那么 `nums[i]` 当然是你当下第一个合法的选择！因此，循环变量 `j` 必须从 `i` 开始，即 `for j in range(i, len(nums))`。

优化点1：这里可以优化成一行```dfs(j, t+candidates[j])```.这里把整数 `t` 当成了像 `path` 列表一样的对象去“恢复现场”。但实际上，**在 Python 中，整数是不可变对象，而列表是可变对象。**`path` 是一个全程共用的列表，你在深层递归里修改了它，回到浅层时必须 `pop()` 把它改回来。`t` 是通过参数传递的局部变量。你完全没必要修改当前层的 `t`，**只需要把计算好的新值直接传给下一层就可以了**。下一层有它自己的 `t`，不会影响当前层的 `t`。

~~~python
        candidates.sort() # 先排序
        ans = []
        path = []

        def dfs(i, t):
            if t == target:
                ans.append(path.copy())
                return 
            #if t > target:  # 删除这行
            #   return 
            if t + candidates[i] > target:   # 加上这个判断
                return 
            for j in range(i, len(candidates)):
                path.append(candidates[j])
                t += candidates[j]
                dfs(j, t)
                t -= candidates[j]
                path.pop()
        dfs(0, 0)
        return ans
~~~

## [79. 单词搜索](https://leetcode.cn/problems/word-search/)

定义一个set集合，存储当前路径中的横纵坐标。

dfs(r, c, i)代表当前到了（r，c）坐标处，需要判断第i个字母能否在这儿落下。

判断他能否落下，要看r和c是否没越界，（r，c）处的坐标是否等于第i个字母，这个横纵坐标是否已经在集合中了（因为同一个单元格内的字母不允许被重复使用）

如果能落下，path中加入该坐标，然后就要向四周做dfs了，如果四周dfs都失败了，说明这个点还是不符合要求，因此需要remove这个点恢复现场。注意是remove，python中集合不能用pop，他会随机删除一个元素，只能用remove指名道姓让他删除指定的元素。

本题最后需要遍历整张图，看看整张图每个节点是否能得到正确的答案。

~~~python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        m, n = len(board), len(board[0])
        path = set()

        def dfs(r, c, i):
            if i == len(word):
                return True
            if r < 0 or r >= m or c < 0 or c >= n or board[r][c] != word[i] or (r, c) in path:
                return False

            path.add((r, c))

            res = (dfs(r, c + 1, i + 1) or
                   dfs(r, c - 1, i + 1) or
                   dfs(r + 1, c, i + 1) or
                   dfs(r - 1, c, i + 1))
            
            path.remove((r, c))
            return res
        
        for r in range(m):
            for c in range(n):
                if dfs(r, c, 0):
                    return True
        return False
~~~

~~~py
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:

        def dfs(i, j, index):
            if index == len(word):
                return True
            if i < 0 or j < 0 or i >= m or j >= n or labels[i][j] or board[i][j] != word[index]:
                return False
            
            labels[i][j] = True
            if dfs(i+1, j, index+1) or dfs(i-1, j, index+1) or dfs(i, j+1, index+1) or dfs(i, j-1, index+1):
                return True
            labels[i][j] = False
            return False
        for i in range(m):
            for j in range(n):
                if dfs(i, j, 0):
                    return True
        return False
~~~

## [131. 分割回文串](https://leetcode.cn/problems/palindrome-partitioning/)

视频链接：https://www.bilibili.com/video/BV1mG4y1A7Gu?vd_source=0a6474eb7d1934754444cf6bf514a053&spm_id_from=333.788.videopod.sections

1. 判断s是否是回文串，有两种方法，一种是用左右双指针往中间夹逼；另一种是看s是否等于s[::-1]
2. 本题可以看作是每个字母之间有挡板，然后选择符合题意得挡板放置情况，选所有挡板得子集
3. 子集型问题，依旧两种做法

**方法1**  首先是第二种思路吧，我觉得更容易一点。**但是这个问题我建议不要当作隔板问题来思考了就当作正常得一道题来思考**。

1. dfs（i）代表下标i之前（不包括下标i）的不用管了，之前得所有字母已经分割成回文串了。要考虑下标i可以和后面的哪几个元素构成回文串，如果和下标j之间的串构成了回文串，就可以dfs(j+1)了，同时注意结束了恢复现场
2. 边界条件是当i == n了，证明所有的字母都处理完了，现在的结果一定是满足题意得，因此可以直接添加了

~~~py
class Solution:
    def partition(self, s: str) -> List[List[str]]:
        if s == None:
            return []
        ans = []
        path = []
        n = len(s)
        def dfs(i):
            if i == n:
                ans.append(path.copy())
                return 
            
            for j in range(i, n):
                t = s[i:j+1]
                if t == t[::-1]:
                    path.append(t)
                    dfs(j+1)
                    path.pop()
        dfs(0)
        return ans
~~~

**方法2**   回溯型子集的第一种思路。我感觉更复杂一些

`dfs(i, start)` 代表当前正在考虑**是否要在下标 `i` 的字符后面放一块“隔板”来进行分割**。其中 `start` 记录的是当前这段还没被分割的子串的起始位置。 在每一个下标 `i` 处，我们面临两个分支选择：

- **不放隔板（不分割）：** 前提是还没走到字符串末尾（`i < n - 1`），说明当前的子串还可以继续往后合并拉长。此时子串起点不变，继续去考察下一个字符，即进入 `dfs(i+1, start)`。
- **放隔板（进行分割）：** 我们把从 `start` 到 `i` 的当前子串 `t = s[start:i+1]` 截取下来。如果 `t` 是回文串，说明符合条件，可以在这里切一刀！我们把 `t` 加入 `path`，然后继续去考虑后续字符，此时**新一段子串的起点就变成了 `i+1`**，即进入 `dfs(i+1, i+1)`。同样，结束当前分支后要注意恢复现场（`path.pop()`）。

边界条件是当 `i == n` 时，证明我们已经考察完了所有的字母缝隙。因为只有遇到合法的回文串时我们才会真正执行分割并把 `start` 往后推，所以只要能一路顺利走到 `i == n`（也就是 `start` 也等于 `n` 时被传入），就说明前面的字母已经完美分割成了回文串，现在的 `path` 一定是满足题意的，因此可以直接 `append` 到结果里。

~~~python
class Solution:
    def partition(self, s: str) -> List[List[str]]:
        if s == None:
            return []
        ans = []
        path = []
        n = len(s)
        def dfs(i, start):
            if i == n:
                ans.append(path.copy())
                return 
            
            if i < n-1:
                dfs(i+1, start)
            
            t = s[start:i+1]
            if t == t[::-1]:
                path.append(t)
                dfs(i+1, i+1)
                path.pop()
        dfs(0, 0)
        return ans
~~~

~~~python
    def solveNQueens(self, n: int) -> List[List[str]]:
        def divert(path):
            res = []
            for i in path:
                res_row = '.' * i + 'Q' + '.' * (n - i - 1)
                res.append(res_row)
            return res
        ans = []
        labels = [False] * n

            if i == n:
                ans.append(divert(path))
                return

                if not labels[j]:
                    flag = True

                            flag = False
                            break

                        labels[j] = False
~~~

优化：可以单独列两个集合set，用来判断，正反两个不同的对角线上是否已经有皇后了。集合set删除某个元素使用的是remove

~~~py
class Solution:
    def solveNQueens(self, n: int) -> List[List[str]]:
        ans = []
        path = []
        
        # 使用三个 Set 来实现 O(1) 级别的冲突检查
        cols = set()       # 记录被占用的列
        diag1 = set()      # 记录被占用的主对角线 (row - col)
        diag2 = set()      # 记录被占用的副对角线 (row + col)

        def dfs(row):
            if row == n:
                # 转化棋盘格式
                board = ["." * col + "Q" + "." * (n - col - 1) for col in path]
                ans.append(board)
                return
            
            for col in range(n):
                # 如果所在的 列、主对角线、副对角线 已经被占用了，直接跳过 (O(1) 时间)
                if col in cols or (row - col) in diag1 or (row + col) in diag2:
                    continue
                
                # 做出选择，记录状态
                cols.add(col)
                diag1.add(row - col)
                diag2.add(row + col)
                path.append(col)
                
                # 进入下一行
                dfs(row + 1)
                
                # 撤销选择，恢复现场
                cols.remove(col)
                diag1.remove(row - col)
                diag2.remove(row + col)
                path.pop()
                
        dfs(0)
        return ans
~~~

## [155. 最小栈](https://leetcode.cn/problems/min-stack/)

将一个普通的栈改成压栈的时候同步压入当前前缀的最小值。也就是可以压入一个元组，元组第一个是数值，第二个是前缀最小值

1. init函数需要加入一个栈底节点，主要是加入一个inf的前缀最大值，这个元组第一个数字无所谓
2. push函数就是加入val的同时判断这个值和前一个值的min谁更小，作为前缀最小值插入
3. pop函数就是普通的pop
4. top函数就是返回最后一个元素，然后返回这个元组的第一个元素
5. getMin函数就是返回最后一个元素，然后返回这个元组的第二个元素

~~~python
class MinStack:

    def __init__(self):
        self.stack = [(inf, inf)]
        

    def push(self, val: int) -> None:
        self.stack.append((val, min(self.getMin(), val)))
        

    def pop(self) -> None:
        self.stack.pop()
        

    def top(self) -> int:
        return self.stack[-1][0]
        

    def getMin(self) -> int:
        return self.stack[-1][1]
~~~

以 *s*=ab2[cd]3[e] 为例说明，将其视作 ab+2[cd]+3[e]。从左到右遍历 *s*：

1. *s*0=a，加到答案中，现在答案为 a。
2. *s*1=b，加到答案中，现在答案为 ab。
3. *s*2=2，更新重复次数 *k*=2。
4. *s*3=[，往下递归。
5. *s*4=c，在下一级递归函数中的字符串为 c。
6. *s*5=d，在下一级递归函数中的字符串为 cd。
7. s6=]，递归返回 cd。上层递归函数接收到 cd，将其重复 k=2 次，得到 cdcd，加到答案中，现在答案为 abcdcd。然后**重置 k=0**。⚠注意：如果不重置，后面遍历到数字 3 的时候，会把 3 加到 2 的后面，得到错误的 k=23。
8. *s*7=3，更新重复次数 *k*=3。
9. *s*8,9,10=[e]，和上面一样，递归，得到字符串 e。将其重复 *k*=3 次，得到 eee，加到答案中，最终答案为 abcdcdeee。

~~~python
class Solution:
    def decodeString(self, s: str) -> str:
        i = 0

        def decode() -> str:
            nonlocal i
            res = ''  # 或者 res = []，具体见另一份代码【Python3 列表】
            k = 0
            while i < len(s):
                c = s[i]
                i += 1
                if c.isalpha():
                    res += c
                elif c.isdigit():
                    k = k * 10 + int(c)
                elif c == '[':  # 递
                    res += decode() * k  # 把括号内的字符串重复 k 次
                    k = 0  # 重置 k，若不重置，2[a]3[b] 后面的 3 会算出 k = 23
                else:  # ']' 归
                    break
            return res

        return decode()
~~~

**方法3**：第二种递归写法用栈模拟的写法

~~~py
class Solution:
    def decodeString(self, s: str) -> str:
        stack = []  # 用于模拟计算机的递归
        res = ''
        k = 0
        for c in s:
            if c.isalpha():
                res += c
            elif c.isdigit():
                k = k * 10 + int(c)
            elif c == '[':
                # 模拟递归
                # 在递归之前，把当前递归函数中的局部变量 res 和 k 保存到栈中
                stack.append((res, k))
                # 递归，初始化 res 和 k
                res = ''
                k = 0
            else:  # ']'
                # 递归结束，从栈中恢复递归之前保存的局部变量
                pre_res, pre_k = stack.pop()
                # 此时 res 是下层递归的返回值，将其重复 pre_k 次，拼接到递归前的 pre_res 之后
                res = pre_res + res * pre_k
        return res
~~~

    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        n = len(temperatures)
        ans = [0] * n
        st = []
        for i in range(n):
            t = temperatures[i]
            while st and t > temperatures[st[-1]]:
                top = st.pop()
                ans[top] = i - top
            st.append(i)
        return ans
~~~

~~~py
from math import inf

        left = [-1] * n
        right = [n] * n

        ans = -inf
        for i, h in enumerate(heights):
            while st and h < heights[st[-1]]:
                j = st.pop()
                right[j] = i
            if st:
                left[i] = st[-1]
            st.append(i)
~~~

## [215. 数组中的第K个最大元素](https://leetcode.cn/problems/kth-largest-element-in-an-array/)

直接用heapq超时的，因为heapq处理一个列表是o（nlogn）的。

这题用类似快排的思想，一直交换然后二分做。

每个快排函数中，可以将随机选中的哨兵节点放到最左边，以防止后面节点的影响。

本题主要看灵神的解答（灵神的解答中也有关于各个大于小于是否取等的说明，可以到时候看看）

~~~python
#这是我自己写的灵神的代码版本
class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        def kuaipai(left, right):
            shaobing = randint(left, right)
            pivot = nums[shaobing]
            nums[shaobing], nums[left] = nums[left], nums[shaobing]
            i, j = left + 1, right
            while True:
                while i <= j and nums[i] < pivot:
                    i += 1
                while i <= j and nums[j] > pivot:
                    j -= 1
                if i >= j:
                    break
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
                j -= 1
            nums[left], nums[j] = nums[j], nums[left]
            
            return j
        left, right = 0, len(nums) - 1
        while True:
            t = kuaipai(left, right)
            if t == len(nums) - k:
                return nums[t]
            if t < len(nums) - k:
                left = t + 1
            else:
                right = t - 1
~~~

> 我们先确立一个**大前提**：划分（Partition）结束时，我们要把最左边的哨兵 `nums[left]` 和 `nums[j]` 交换。**这意味着，循环结束时，`j` 指向的元素必须小于或等于哨兵（`<= pivot`）**，否则交换后，大于哨兵的元素就被换到了左边，整个算法就毁了。
>
> 让我们逐一破解这些“为什么”。
>
> ------
>
> ### 1. 为什么是 `while i <= j` 而不是 `i < j`？
>
> **核心原因：为了让指针交错（`i > j`），从而保证 `j` 停在正确的位置。**
>
> 假设我们使用 `while i < j`:
>
> - 当 `i` 和 `j` 相遇时（`i == j`），外层和内层循环都会停止。
> - 但此时，相遇位置的元素**还没有和 `pivot` 进行比较**！
>
> **举个致命的例子：**
>
> 数组：`[4, 2, 8, 5]`，哨兵 `pivot = 4`。
>
> - 初始：`i = 1` (指向 2)，`j = 3` (指向 5)。
> - `nums[i] < 4`，`i` 右移变成 2 (指向 8)。此时 `i < j` 成立。
> - `nums[j] > 4`，`j` 左移变成 2 (指向 8)。
> - 此时 `i == j == 2`。如果条件是 `i < j`，循环直接结束。
> - 最后执行收尾交换：`nums[left], nums[j] = nums[j], nums[left]`。也就是把 `nums[0]`(4) 和 `nums[2]`(8) 交换。
> - 数组变成 `[8, 2, 4, 5]`。**错误！8 比 4 大，却跑到了 4 的左边！**
>
> **使用 `while i <= j` 的效果：**
>
> 当 `i == j == 2`（指向 8）时，循环继续！
>
> - 内层判定：`nums[j] > pivot` (8 > 4) 成立，所以 `j -= 1`，`j` 退回到了 1（指向 2）。
> - 此时 `i = 2, j = 1`，`i > j`，触发 `if i >= j: break`。
> - 收尾交换：`nums[0]`(4) 和 `nums[1]`(2) 交换，数组变成 `[2, 4, 8, 5]`。**完美！**
>
> ------
>
> ### 2. 为什么是 `nums[i] < pivot` 而不是 `nums[i] <= pivot`？
>
> ### （同理，为什么是 `nums[j] > pivot` 而不是 `>=`？）
>
> **核心原因：为了应对包含大量重复元素的极端情况，防止时间复杂度退化为 $O(N^2)$。**
>
> 假设我们用的是 `nums[i] <= pivot` 和 `nums[j] >= pivot`。
>
> **举个致命的例子：**
>
> 数组全是相同的元素：`[5, 5, 5, 5, 5]`，哨兵 `pivot = 5`。
>
> - 如果带有等号：`nums[i] <= 5` 永远成立。指针 `i` 会像脱缰的野马一直往右跑，直到越界。
> - 这导致所有的元素都被划分到了哨兵的左边，右边一个都没有。
> - 每次划分只能排好一个元素（哨兵自己），递归树变成一条斜线，时间复杂度直接退化成 $O(N^2)$，在 LeetCode 上会直接 **超时 (Time Limit Exceeded)**。
>
> **使用严格的 `<` 和 `>` 的效果：**
>
> 同样是 `[5, 5, 5, 5, 5]`。
>
> - 因为 `nums[i] < 5` 不成立，`i` 停在下标 1。
> - 因为 `nums[j] > 5` 不成立，`j` 停在最后。
> - 此时 `i` 和 `j` 指向的虽然都是 5，但依然会执行交换（5 和 5 换），然后 `i += 1`, `j -= 1`。
> - 这样做的精妙之处在于：**强行让 `i` 和 `j` 在数组的中间相遇**。
> - 哨兵被放在了数组正中间，每次都能将数组完美对半劈开，即使全是重复元素，时间复杂度依然稳稳保持在 $O(N)$！
>
> ------
>
> ### 3. 为什么 break 条件是 `i >= j` 而不是 `i == j`？
>
> **核心原因：`i` 和 `j` 的步伐会导致它们直接“跨过”彼此。**
>
> 有两个地方会让 `i` 和 `j` 发生变化：
>
> 1. 内层的 `while` 循环里，`i += 1` 或 `j -= 1`。
> 2. 找到需要交换的元素后，最后统一执行的 `i += 1` 和 `j -= 1`。
>
> 当最后一次交换发生后，`i` 和 `j` 各自往前/后走一步，此时极有可能出现 `i = j + 1` 的情况（即 `i > j`）。
>
> 如果你的跳出条件仅仅是 `if i == j`，代码就会错过这个跳出时机，进入死循环或者数组越界。使用 `i >= j` 是最安全的兜底策略，只要指针交错，就立刻停止。
>
> ------
>
> ### 总结口诀
>
> - **`i <= j` 允许交错**：为了保证 `j` 最后指向的一定是安全的（`<= pivot`）元素，不留死角。
> - **不加等号 `<` 和 `>`**：为了在遇到大量重复元素时，让左右指针均匀停止并发生看似无用的交换，从而保证划分居中，保住 $O(N)$ 的效率。
> - **`i >= j` 退出**：应对指针交错的情况，防止死循环。

~~~python
class Solution:
    def rob(self, nums: List[int]) -> int:
        n = len(nums)
        @cache
        def dfs(i):
            if i < 0:
~~~

## [322. 零钱兑换](https://leetcode.cn/problems/coin-change/)

完全背包问题。灵神题解https://www.bilibili.com/video/BV16Y411v7Y6?vd_source=0a6474eb7d1934754444cf6bf514a053&spm_id_from=333.788.videopod.sections

递推式为：`dfs(i, amount) = min(dfs(i-1, amount), dfs(i, amount-coins[i]) + 1)`。注意第二个是用了当前的硬币，所以他的硬币数需要加1。

**本题重要的**是边界公式，首先是判断i到了边界，也即i<0了，这时判断c是否=0，如果是就可以返回0，否则返回最大值（因为本题是求最小值）。

然后是判断如果当前金额的边界，如果他小于了当前硬币面额，就不能走第二个支路了，直接往前走就行了。

~~~python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        @cache
        def dfs(i, a):
            if i < 0:
                return 0 if a == 0 else inf
            if a < coins[i]:
                return dfs(i-1, a)
            return min(dfs(i-1, a), dfs(i, a-coins[i]) + 1)
        res = dfs(len(coins)-1, amount)
        return res if res != inf else -1
~~~

转成递推

`dp[i+1][a] = min(dp[i][a], dp[i+1][a-coins[i]] + 1)`

同样的，本题注意的主要就是边界值的情况。以及还要关注dp初始化的时候，amount因为是从0到amount都有，所以需要amount+1，行数是n+1，是因为我们的递推公式模仿灵神把i都加了个1。

~~~py
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        n = len(coins)
        dp = [[inf] * (amount+1) for _ in range(n+1)]
        dp[0][0] = 0
        for i, x in enumerate(coins):
            for a in range(amount+1):
                if a < x:
                    dp[i+1][a] = dp[i][a]
                else:
                    dp[i+1][a] = min(dp[i][a], dp[i+1][a-x]+1)
        ans = dp[n][amount]
        return ans if ans < inf else -1
~~~

~~~python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        n = len(coins)
        dp = [inf] * (amount+1)
        dp[0] = 0
        for i, x in enumerate(coins):
            for a in range(x, amount+1):
                dp[a] = min(dp[a], dp[a-x]+1)
        ans = dp[amount]
        return ans if ans < inf else -1
~~~

~~~python
    def minDistance(self, word1: str, word2: str) -> int:
~~~

~~~py
        for i in range(n1+1):
            f[i][0] = i

            f[0][j] = j

                    f[i+1][j+1] = f[i][j]
                else:
                    f[i+1][j+1] = min(f[i][j], f[i+1][j], f[i][j+1]) + 1
        return f[n1][n2]
~~~

## [136. 只出现一次的数字](https://leetcode.cn/problems/single-number/)

> ### 1. 两个整数的异或（XOR）到底代表什么？
>
> 在计算机底层，整数都是以**二进制**形式存储的。异或（符号通常表示为 `^` 或 $\oplus$）是一种**位运算**，它会对两个整数的每一位（bit）逐一进行比较。
>
> **核心规则就一句话：相同为 0，不同为 1。**
>
> - $0 \oplus 0 = 0$
> - $1 \oplus 1 = 0$
> - $1 \oplus 0 = 1$
> - $0 \oplus 1 = 1$
>
> **通俗的理解：异或是“无进位的二进制加法”**
>
> 你可以把它想象成在做二进制加法，但是**不进位**。比如 $1 + 1$ 本来应该是 $10$（即进位 $1$，当前位留 $0$），如果在异或的世界里，丢弃进位，结果就是 $0$。
>
> **具体例子计算（例如 `3 ⊕ 5`）：**
>
> - `3` 的二进制是：`0011`
>
> - `5` 的二进制是：`0101`
>
> - 上下对齐，逐位异或：
>
>   `0011`
>
>   `0101`
>
>   `----`
>
>   `0110`  (这就是十进制的 `6`)
>
>   所以，$3 \oplus 5 = 6$。
>
> ------
>
> ### 2. 异或运算满足哪些数学定律？
>
> 像加法和乘法一样，异或运算也是有严谨的代数性质的。
>
> #### ✅ 交换律 (Commutative Law)
>
> **满足。** 两个数异或，前后顺序不影响结果。
>
> - 公式：$a \oplus b = b \oplus a$
> - 这就好比“你不等于我，就意味着我不等于你”，比较的双方是对等的。
>
> #### ✅ 结合律 (Associative Law)
>
> **满足。** 多个数异或，先算哪一部分都不影响最终结果。（这就是你在上面那个例子中，能把相同的数字用括号括起来先算的原因）。
>
> - 公式：$(a \oplus b) \oplus c = a \oplus (b \oplus c)$
>
> #### ⚠️ 分配律 (Distributive Law)
>
> **视情况而定。** 分配律通常涉及**两种**不同的运算符（比如乘法对加法的分配律 $a \times (b + c) = a \times b + a \times c$）。对于异或运算：
>
> 1. **与（AND, `&`）运算对异或（XOR, $\oplus$）满足分配律：**
>    - 公式：$a \ \& \ (b \oplus c) = (a \ \& \ b) \oplus (a \ \& \ c)$
>    - （在布尔代数中，AND 相当于乘法，XOR 相当于加法，这就是非常经典的“乘法对加法的分配律”）。
> 2. **或（OR, `|`）运算对异或（XOR, $\oplus$）不满足分配律。**
> 3. **异或自己对其他逻辑运算（如 AND / OR）也不具备分配性质。**
>
> ------
>
> ### 3. 异或最常用的三大“杀手锏”性质
>
> 总结一下，除了你提到的性质，这三条公式是你以后做算法题经常会用到的：
>
> 1. **归零律：** $a \oplus a = 0$ （两个相同的数异或，结果为 0。这就是“消除”的基础）
> 2. **恒等律：** $a \oplus 0 = a$ （任何数和 0 异或，保留原值。这就是你代码里 `ans=0` 可以作为起点的理由）
> 3. **自反律：** $a \oplus b \oplus b = a$ （一个数被同一个数异或两次，等于变回自己。常用于简单的加密解密，或者不借助第三个变量交换两个数的 值）

你可以把这个过程想象成一场“诸侯争霸”：

- 数组里的每一个数字代表一个国家的士兵。
- 因为“多数元素”的定义是出现次数**大于** `⌊ n/2 ⌋`，这意味着**多数元素的士兵数量，比其他所有国家士兵数量的总和还要多**。
- 核心战术就是**“同归于尽”**：如果两个士兵不是同一个国家的，他们就互拼双双阵亡。
- 因为多数元素的兵力绝对占优，就算其他所有国家联合起来跟它“极限一换一”，最后活下来的、站在战场上的绝对是多数元素的士兵。

~~~python
class Solution:
    def majorityElement(self, nums: list[int]) -> int:
        count = 0
        candidate = None

        for num in nums:
            # 如果当前没有候选人（或者兵力消耗殆尽），当前数字就成为新的候选人
            if count == 0:
                candidate = num
            
            # 如果遇到的是同阵营的，兵力加 1；如果是不同阵营的，兵力减 1（一换一）
            if num == candidate:
                count += 1
            else:
                count -= 1
                
        return candidate
~~~

我的思路是设置了两个值，分别代表红色的最后一个位置的下一个，白色的最后一个位置的下一个。
![image-20260325160201114](C:\Users\29743\AppData\Roaming\Typora\typora-user-images\image-20260325160201114.png)

但是这么做不对，看ai解析如下

> ### 🐞 问题所在：重叠索引的多元赋值陷阱
>
> 仔细看你的这行三元交换代码：
>
> Python
>
> ```
> nums[i], nums[red_0], nums[white_1] = nums[white_1], nums[i], nums[red_0]
> ```
>
> 假设现在数组刚开始 `nums = [2, 0, ...]`，此时 `red_0 = 0`, `white_1 = 0`。 当遍历到 `i = 1`，遇到数字 `0` 时：
>
> 1. Python 会先计算等号**右边**的值：`(nums[0], nums[1], nums[0])`，也就是 `(2, 0, 2)`。
> 2. 然后 Python 会从左到右依次执行**左边**的赋值操作：
>    - `nums[1] = 2` （没问题）
>    - `nums[0] = 0` （把 0 放到了红色的位置，没问题）
>    - `nums[0] = 2` （**出大问题了！**因为 `white_1` 和 `red_0` 都是 0，这一步把刚刚好不容易排好的 0 又给覆盖成了 2！）
>
> 因为 `red_0` 和 `white_1` 在最开始或者后续过程中很可能是**同一个索引**，在同一行代码里对同一个索引赋值两次，后一次会覆盖前一次，这就导致原本的 `0` 凭空消失了。
>
> ------
>
> ### ✅ 修正后的代码（保留你的原思路）
>
> 为了避免这个陷阱，我们只需要把交换动作拆开。当你遇到 `0` 的时候： 先把它换到 `red_0` 的位置。但是如果此时 `white_1 > red_0`，说明你刚才把一个本来排好的 `1` 给换到了 `i` 的位置，所以你需要**再补一次交换**，把这个 `1` 换到 `white_1` 的位置上去。

经过ai修正后我的思路的代码是这样的
~~~python
from typing import List

class Solution:
    def sortColors(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        red_0, white_1 = 0, 0
        
        for i in range(len(nums)):
            if nums[i] == 1:
                # 遇到 1，直接和 white_1 交换，白色边界扩张
                nums[i], nums[white_1] = nums[white_1], nums[i]
                white_1 += 1
                
            elif nums[i] == 0:
                # 遇到 0，先把它换到红色的地盘
                nums[i], nums[red_0] = nums[red_0], nums[i]
                
                # 如果白色边界已经超过了红色边界，说明刚才被换到 i 位置的是个 1
                # 我们需要把它再次换到白色的地盘
                if red_0 < white_1:
                    nums[i], nums[white_1] = nums[white_1], nums[i]
                    
                # 红色和白色边界同时扩张
                red_0 += 1
                white_1 += 1
~~~

但是还有更简单的算法：**荷兰国旗算法**

我们设定三个指针：

- **`p0` (指向红色 0 的右边界)：** 它的左边（不含它自己）全是 `0`。它代表着下一个 `0` 应该被放置的位置。
- **`p2` (指向蓝色 2 的左边界)：** 它的右边（不含它自己）全是 `2`。它代表着下一个 `2` 应该被放置的位置。
- **`curr` (当前遍历指针)：** 像一个侦察兵，从左往右扫描数组中的每一个未知的数字。

这三个指针把整个数组切分成了四个动态变化的区域：

1. `[0, p0 - 1]`：全是 **0**
2. `[p0, curr - 1]`：全是 **1**
3. `[curr, p2]`：**未知区域**（还没遍历到的数字）
4. `[p2 + 1, len - 1]`：全是 **2**

当 `curr` 走过 `p2`（即 `curr > p2`）时，未知区域被全部探索完毕，排序就完成了！

~~~python
from typing import List

class Solution:
    def sortColors(self, nums: List[int]) -> None:
        p0 = 0
        curr = 0
        p2 = len(nums) - 1
        
        # 只要侦察兵还没有越过 p2 的防线，就继续探索
        while curr <= p2:
            if nums[curr] == 0:
                # 遇到 0，把它扔到前面 p0 的位置，然后两个指针一起前进
                nums[curr], nums[p0] = nums[p0], nums[curr]
                p0 += 1
                curr += 1
                
            elif nums[curr] == 2:
                # 遇到 2，把它扔到后面 p2 的位置
                nums[curr], nums[p2] = nums[p2], nums[curr]
                # 注意：这里 p2 往左缩减，但 curr 必须原地不动！
                p2 -= 1
                
            else:
                # 遇到 1 (nums[curr] == 1)，最省事，直接跳过
                curr += 1
~~~

> ###  为什么这样是合理的？（最容易困惑的两个点）
>
> 这套逻辑初看很简单，但有两个地方经常让人想不通。弄懂了这两个问题，你就彻底掌握了这个算法的灵魂：
>
> **1. 为什么遇到 `2` 时，和 `p2` 交换后，`curr` 必须原地不动？** 因为你把 `curr` 上的 `2` 扔到了后面，同时把后面 `p2` 位置上的数字换到了前面。**从 `p2` 换过来的这个新数字，是来自“未知区域”的。** 侦察兵 `curr` 还没检查过它！它可能是个 `0`，也可能是个 `1`，甚至还是个 `2`。所以 `curr` 必须停在原地，在下一次循环中重新鉴定这个新换过来的数字。
>
> **2. 那为什么遇到 `0` 时，和 `p0` 交换后，`curr` 就可以放心地 `+1` 前进呢？** 这是一个绝妙的推论！因为 `curr` 是从左往右走的，它永远走在 `p0` 的前面（或者重合）。
>
> - `curr` 走过的地方，所有的 `2` 都已经被扔到数组尾部去了。
> - 所有的 `0` 都已经被扔到 `p0` 及其左边了。
> - 所以，**`p0` 所指向的位置，如果有数字，那它一定只可能是 `1`（或者 `curr` 和 `p0` 重合时指向同一个 `0`）。**
>
> 因此，当你把 `curr` 指向的 `0` 换到 `p0` 时，从 `p0` 换回来的那个数字**绝对安全**（100% 是个 `1`）。既然换回来的是个老实本分的 `1`，`curr` 就可以放心地继续往前走，不用再停下来检查了。

## [31. 下一个排列](https://leetcode.cn/problems/next-permutation/)

这个题看**灵神的题解**，感觉讲的很棒

这个题实际意思就是看当前这个整数的下一个整数是什么?

三步：

第一步从右往左遍历，找到第一个小于他右边数字的点。记为K。那么K右边就是单调递减的。

第二步从右往左遍历，找到第一个大于K的点，将这个点的值和位置K的值做交换

第三步，现在K后面的还是单调递减的，需要reverse换成单调递增

此外，临界条件需要特判，比如原来整个数字就是单调递减的，K就是-1，这个时候直接reverse整个数组就行

## [287. 寻找重复数](https://leetcode.cn/problems/find-the-duplicate-number/)

我刚开始写了一份会修改原来数组的代码

~~~python
from typing import List

class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        i = 0
        while True:
            # 1. 如果当前数字已经在正确的索引上，继续下一个
            if nums[i] == i:
                i += 1
                continue
                
            # 修改一：将 nums[i] < i 改为判断“目标位置是不是已经有相同的数了”
            if nums[i] == nums[nums[i]]:
                return nums[i]
                
            # 修改二：删去内层的 while 循环，因为外层已经是 while True 了，直接交换即可
            temp = nums[i]
            nums[i], nums[temp] = nums[temp], nums[i]
            
            # 修改三：删掉这里的 i += 1 （这是导致越界的罪魁祸首）
~~~

## [110. 平衡二叉树](https://leetcode.cn/problems/balanced-binary-tree/)

~~~python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        def get_height(node):
            if not node:
                return 0
            l_height = get_height(node.left)
            if l_height == -1:
                return -1
            r_height = get_height(node.right)
            if r_height == -1:
                return -1
            if abs(l_height - r_height) > 1:
                return -1
            return max(l_height, r_height) + 1
        return get_height(root) != -1
~~~

这里是灵神的方法，用-1来表示这个树已经不是平衡二叉树了，所以可以直接往上走了，做一个剪枝。
